---
name: explain-code
description: "Explains code with visual diagrams and analogies.  Use when explaining how code works, teaching about a codebase, or when the user asks 'how does this work?'"
---

When explaining code, always include:

1. **Start with an analogy**: Compare the code to something from everyday life
2. **Draw a diagram**: Use ASCII art to illustrate the structure and flow
3. **Explain line by line**: Walk through the code in detail, connecting it back
4. **Highlight a gotcha**: What's a common mistake or misconception about this code?

Keep explanations conversational. For complex concepts, use multiple analogies or diagrams to clarify. Always check for understanding before moving on.
